FeynMF Reference Manual

Files and Diagrams

\begin{fmffile}{file} LaTeX code \end{fmffile}
Write all diagrams in LaTeX code into the file file.
\begin{fmfgraph*}(width,height) FeynMF commands \end{fmfgraph*}
Construct a Feynman diagram of a given width and height (measured in units of \unitlength), specified by the FeynMF commands.
\begin{fmfgraph}(width,height) FeynMF commands \end{fmfgraph}
A slightly more efficient version of fmfgraph*, which does not support labels for vertices and arcs.
\fmfframe(left,top)(right,bottom){box}
Put an (invisible) frame around box. The size of the frame is given by the dimensions left, top, right and bottom. This is useful if labels are overshooting the box defined by the fmfgraph* environment.

\fmfinit
\fmffreeze
Calculate the positions of the vertices based on the arcs which are defined up to this point. Usually this calculation is performed automatically at the end of the fmfgraph environment. Calling it explicitely is useful for later adding arcs that should not enter the calculation . The layout is chosen to minimize the overall length of all arcs. The length of each arc is weighted with the tension option, whose default value is 1.

Pick up a pen of the size #1. This is used for changing the weight (i.e. thickness) of the lines. Predefined sizes are thin and thick.

Tips and Tricks

Making Code Work with Older or Newer FeynMF Versions

FeynMF versions before 1.03 required the use of \noexpand before certain TeX macros in labels. Here \bar is the most notorious example. Since version 1.03 this is no longer necessary. In fact, since version 1.05 it is no longer allowed because the compatibility code can trigger unwanted side effects, which are hard to track down.

Fortunately, it is possible to make documents portable to installations which are still running old versions of FeynMF by conditionally defining \nxp (any other name could be used as well)

  \expandafter\ifx\csname fmfpfx\endcsname\relax
    \let\nxp\noexpand  % version <= 1.02: halt expansion \else \let\nxp\relax % version>= 1.03: hacks not needed anymore
  \fi

and using it in place of \noexpand

  \fmf{plain,label=$\nxp\bar p$}{from,to}

Usually, it is possible to process documents prepared for old versions of FeynMF by using the pre-1.03 option:

  \usepackage[pre-1.03]{feynmf}

But as mentioned above, this can interfere with the LaTeX font loading on demand code and should be used with care.
